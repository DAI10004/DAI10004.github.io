e:T941,<h1>Next.js 静态站点生成（SSG）详解</h1>
<p>静态站点生成（Static Site Generation, SSG）是 Next.js 最重要的特性之一。</p>
<h2>什么是 SSG？</h2>
<p>SSG 在<strong>构建时</strong>（build time）生成 HTML 页面，而不是在每次请求时生成。这意味着：</p>
<ul>
<li>页面在部署时就已经生成好</li>
<li>用户请求时直接返回静态 HTML</li>
<li>无需服务器端运行时</li>
</ul>
<h2>SSG 的优势</h2>
<h3>1. 性能优秀</h3>
<ul>
<li>静态 HTML 可以被 CDN 缓存</li>
<li>首屏加载速度极快</li>
<li>SEO 友好</li>
</ul>
<h3>2. 稳定性高</h3>
<ul>
<li>无需数据库</li>
<li>无需后端服务器</li>
<li>减少故障点</li>
</ul>
<h3>3. 成本低</h3>
<ul>
<li>可以使用 GitHub Pages 等免费托管</li>
<li>无需运行服务器</li>
</ul>
<h2>Next.js 中的 SSG 实现</h2>
<h3>基本用法</h3>
<pre><code class="language-typescript">export default function Page({ data }) {
  return &#x3C;div>{data}&#x3C;/div>
}

// 在构建时获取数据
export async function getStaticProps() {
  const data = await fetchData()
  return {
    props: { data }
  }
}
</code></pre>
<h3>动态路由</h3>
<pre><code class="language-typescript">// pages/blog/[slug].tsx
export async function getStaticPaths() {
  const posts = await getPosts()
  return {
    paths: posts.map(post => ({
      params: { slug: post.slug }
    })),
    fallback: false
  }
}

export async function getStaticProps({ params }) {
  const post = await getPost(params.slug)
  return {
    props: { post }
  }
}
</code></pre>
<h2>何时使用 SSG？</h2>
<p><strong>适合 SSG 的场景：</strong></p>
<ul>
<li>营销页面</li>
<li>文档网站</li>
<li>博客</li>
<li>产品展示页</li>
<li>内容更新不频繁的网站</li>
</ul>
<p><strong>不适合 SSG 的场景：</strong></p>
<ul>
<li>需要用户登录的内容</li>
<li>实时数据</li>
<li>用户个性化内容</li>
<li>频繁更新的内容</li>
</ul>
<h2>本博客的架构</h2>
<p>本博客采用典型的 SSG 架构：</p>
<pre><code>Markdown 文件
    ↓
构建时读取并解析
    ↓
生成静态 HTML
    ↓
部署到 GitHub Pages
</code></pre>
<h2>总结</h2>
<p>SSG 是构建现代静态网站的理想选择，结合了性能、SEO 和开发效率。</p>
<p>如果你的内容在构建时就能确定，SSG 是最佳选择。</p>
